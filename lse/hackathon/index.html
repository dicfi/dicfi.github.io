Reflexiones del Taller de Fabricación y Adaptación de Hardware y Software para la Construcción de Robots Educativos

Durante los dias 29, 30 y 31 de julio, 1 y 2 de agosto de 2013, realizamos un hackathon en la Facultad de Informática. En el marco del Proyecto de Investigación “Identificación, Evaluación y Adaptación de Componentes de Hardware y Software para Sistemas Embebidos”.

El taller consistió primeramente en estudiar la arquitectura de hardware y software de los robots educativos que estamos fabricando y usando.
Luego, durante los dias restantes, se fabricaron y adaptaron los componentes de hardware y software, para la fabricación de tres robots.

Esto incluyó tareas de electrónica, como soldar componentes para la fabricacion de placas easyduino, verificación y debug del funcionamiento electrónico, implantación del firmware en los microcontroladores etc. Además, se realizaron tareas de software en los "cerebros" de los robots (placas con Linux embebidos), tales como instalar OpenWRT Linux en las placas, instalación y configuración de software específico para los robots, utilización del ambiente arduino para compilar y transferir software a los ATMEGA8, etc.

Las tareas de electrónica las hicieron los alumnos de Ingeniería Electrónica, y las de software los alumnos de la Licenciatura en Ciencias de la Computación.

El Viernes se logró hacer los tests de integración, entre las placas de electrónica y los cerebros. Además, se hicieron pruebas de transferencia de software entre los dispositivos, que servian para doble propósito: 

  • Para que la electrónica ya tuviera el software que controla los robots;
  •  y para testear que las placas de electrónica, y las comunicaciones funcionaban. Las partes que se verificaron fueron: PC -> wireless -> placas con Linux embebido -> cablerio serial -> placa electrónica Easyduino.

Finalmente, como para cerrar, se testearon algunos motorcitos en conjunto con las demás partes desarrolladas, para realizar pruebas en alto nivel que "simularan" la programación que utilizamos para controlar los robots (para "moverlo" bah :) ).

Quedamos muy contentos con los logros. Seguramente existen muchas cosas que podemos mejorar y aprender todos, como tarea futura.

Como meta pendiente  (y continuación del trabajo) intentaremos documentar públicamente todo el proceso de construccion de los robots (que partes se necesitan, donde se compran, como se construyen, como se configuran, etc). Para que otras instituciones tengan la posibilidad de armar sus propios robots si quisiesen algo como el nuestro. O para que puedan "vernos" y ayudarnos con mejoras si están mas avanzados.
